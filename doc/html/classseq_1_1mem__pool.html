Detailed Description

template<class T, class Allocator = std::allocator<T>, size_t Align = 0, class Lock = null_lock>
class seq::mem_pool< T, Allocator, Align, Lock >

Memory pool class used to allocate individual elements.

mem_pool class allocates/deallocates individual elements of type T. It internally allocates chunks of T objects using a growing strategy based on the global growth factor SEQ_GROW_FACTOR (1.6 by default). The chunks are allocated using given allocator as template parameter, by default std::allocator<T>.

Objects allocated using #mem_pool::allocate are always aligned on the default platform alignment value (usually alignof(std::max_align_t)), even if sizeof(T) < alignof(std::max_align_t).

The Lock template parameter is a mutex-like class that can be set to std::mutex or seq::spinlock for multithreaded allocations/deallocations. Using mem_pool class in a multithreaded context with the default null_lock class will introduce undefined behaviors in your program.

If ReclaimMemory is false, the mem_pool object will never free its internal memory on calls to #mem_pool::deallocate. In such case, only calls to #mem_pool::release_unused_memory() and the class destructor will release previously allocated memory.

mem_pool class has been tested on Windows and Linux where performances are very similar:

  • Calls to #mem_pool::allocate and mem_pool::deallocate are usually 5 times faster than the stock malloc/free, and around 3 times faster when using seq::spinlock.
  • Using #mem_pool::allocate instead of malloc results in a 8 bytes saving per allocation (on 64 bits platforms), since malloc almost always uses a 8-4 bytes control block per allocation.

template<class T , class Allocator = std::allocator<T>, size_t Align = 0, class Lock = null_lock>
void seq::mem_pool< T, Allocator, Align, Lock >::transfer_content_to ( mem_pool< T, Allocator, Align, Lock > &  other)
inline

Transfer the content of this mem_pool to other. All the slots (occupied or not) ar transferred to other. Therefore, objects previously allocated by this can only be deallocated by other after the call to transfer_content_to. this mem_pool mem_pool is considered as empty and can be reused afterward.

This function does nothing if get_allocator() != other.get_allocator().
