\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Key, class T, class Compare = std\+::less$<$\+Key$>$, class Allocator = std\+::allocator$<$std\+::pair$<$\+Key,\+T$>$ $>$, Layout\+Management layout = Optimize\+For\+Memory, bool Stable = false, bool Unique = true$>$\newline
class seq\+::flat\+\_\+map$<$ Key, T, Compare, Allocator, layout, Stable, Unique $>$}

flat sorted associative container that contains key-\/value pairs with unique keys similar to boost\+::flat\+\_\+map with faster insertion/deletion of single values

\mbox{\hyperlink{classseq_1_1flat__map}{seq\+::flat\+\_\+map}} is a sorted associative container similar to \href{https://www.boost.org/doc/libs/1_64_0/doc/html/boost/container/flat_map.html}{\texttt{ boost\+::flat\+\_\+map}}, but relying on a \mbox{\hyperlink{classseq_1_1deque}{seq\+::deque}} instead of a flat array.

\mbox{\hyperlink{classseq_1_1flat__map}{seq\+::flat\+\_\+map}} behaves like \mbox{\hyperlink{classseq_1_1flat__set}{seq\+::flat\+\_\+set}} except that the underlying deque stores elements of types std\+::pair$<$\+Key,\+T$>$ instead of Key. Its interface is similar to std\+::map, except for the node based members.

All references and iterators are invalidated when inserting/removing elements.

Unlike std\+::map which stores objects of type std\+::pair$<$const Key,\+T$>$, \mbox{\hyperlink{classseq_1_1flat__map}{flat\+\_\+map}} stores objects of type std\+::pair$<$\+Key,\+T$>$. To avoid modifying the key value through iterators or using \mbox{\hyperlink{classseq_1_1flat__map_a686503d3cb7ac516b4f747c0034a547b}{flat\+\_\+map\+::pos()}}, the value is reinterpreted to std\+::pair$<$const Key,\+T$>$. Although quite ugly, this solution works on all tested compilers.
