\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T, class Allocator = std\+::allocator$<$\+T$>$, size\+\_\+t Align = 0, class Lock = null\+\_\+lock$>$\newline
class seq\+::mem\+\_\+pool\+\_\+allocator$<$ T, Allocator, Align, Lock $>$}

Custom allocator based on \mbox{\hyperlink{classmem__pool}{mem\+\_\+pool}} class.

{\itshape \mbox{\hyperlink{classseq_1_1mem__pool__allocator}{mem\+\_\+pool\+\_\+allocator}}} is a stl compliant allocator dedicated to node based containers like std\+::set, std\+::map, std\+::list, std\+::deque, std\+::multimap, std\+::multiset...

It provides faster allocation/deallocation time as well as reduced memory footprint (see \mbox{\hyperlink{classmem__pool}{mem\+\_\+pool}} class for more details). Since the {\itshape \mbox{\hyperlink{classseq_1_1mem__pool}{mem\+\_\+pool}}} class uses chunk based allocation, containers using {\itshape \mbox{\hyperlink{classseq_1_1mem__pool__allocator}{mem\+\_\+pool\+\_\+allocator}}} are usually more cache friendly, which fasten most operations (like iterating, sorting,...).

The {\itshape Allocator} template parameter is forwarded to the underlying \mbox{\hyperlink{classseq_1_1mem__pool}{mem\+\_\+pool}} objects. It is also used when allocating more than one object. Therafore, using {\itshape \mbox{\hyperlink{classseq_1_1mem__pool__allocator}{mem\+\_\+pool\+\_\+allocator}}} does not benefit to containers relying on contiguous memory blocks like std\+::vector.
