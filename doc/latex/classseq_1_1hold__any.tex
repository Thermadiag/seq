\mbox{\hyperlink{classseq_1_1hold__any}{seq\+::hold\+\_\+any}} is a std\+::any like class optimized to create heterogeneous containers of any types (vectors, deques, hash tables, sorted containers...).

The seq library provides the following aliases\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{namespaceseq_a24ef193cef19271b9b18ce6c74fc5b54}{seq\+::any}}\+: equivalent to seq\+::hold\+\_\+any$<$seq\+::any\+\_\+default\+\_\+interface$>$
\item \mbox{\hyperlink{namespaceseq_a2ec98840dd82cf3a3207402caa66c73f}{seq\+::nh\+\_\+any}}\+: equivalent to seq\+::hold\+\_\+any$<$seq\+::any\+\_\+no\+\_\+hash\+\_\+interface$>$
\end{DoxyItemize}\hypertarget{classseq_1_1hold__any_autotoc_md0}{}\doxysubsubsection{Interface}\label{classseq_1_1hold__any_autotoc_md0}
\mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} offers a similar interface to std\+::any and supports additional features\+:
\begin{DoxyItemize}
\item Support for comparison operators $<$, $<$=, $>$, $>$=, == and !=
\item Support for standard stream operators to/from std\+::ostream/std\+::istream
\item \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} is hashable based on std\+::hash
\item \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} can be formatted using the \mbox{\hyperlink{group__format}{formatting}} module of seq library
\item \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} interface can be extended
\end{DoxyItemize}

\mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} is very similar to boost.\+Type\+Erasure or folly.\+Poly, but its default implementation provides more features in order to be used within most containers (sorted containers, hash tables,...).

These features do not modify the requirements of held types\+: they must be at least move only contructible. \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} uses type erasure to provide custom behavior based on held object type, and the additional functions like streaming or comparison support are implemented if the type supports them. If not, the corresponding functions with throw a std\+::bad\+\_\+function\+\_\+call exception\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{std::cout << \mbox{\hyperlink{namespaceseq_a24ef193cef19271b9b18ce6c74fc5b54}{seq::any}}(2);                   \textcolor{comment}{// works as expected, print '2'}}
\DoxyCodeLine{std::cout << seq::any(std::vector<bool>()); \textcolor{comment}{// std::vector<bool> does not define stream operator to std::ostream, throw a std::bad\_function\_call}}
\end{DoxyCode}


The only exception is the hashing support. Indeed, there is now way in C++11 to tell at compile time if a type is hashable or not, as most implementations of std\+::hash use static\+\_\+assert() in the operator() member (undetectable through S\+F\+I\+N\+AE). \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} optimistically assumes that all types are hashable. In order to store non hashable types, you must specialize the \mbox{\hyperlink{structseq_1_1is__hashable}{seq\+::is\+\_\+hashable}} type trait for this type\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{struct }test\{\};}
\DoxyCodeLine{}

\DoxyCodeLine{\textcolor{keyword}{struct }\mbox{\hyperlink{structseq_1_1is__hashable}{seq::is\_hashable}}<test> : std::false\_type \{\};}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{//...}}
\DoxyCodeLine{}
\DoxyCodeLine{\mbox{\hyperlink{classseq_1_1hold__any}{seq::any}} a = test(); \textcolor{comment}{// compile as is\_hashable is specialized for test class}}
\end{DoxyCode}


Otherwise, you can completely disable the hashing support using \mbox{\hyperlink{namespaceseq_a2ec98840dd82cf3a3207402caa66c73f}{seq\+::nh\+\_\+any}} (equivalent to seq\+::hold\+\_\+any$<$seq\+::any\+\_\+no\+\_\+hash\+\_\+interface$>$) \+:

\hypertarget{classseq_1_1hold__any_autotoc_md2}{}\doxysubsubsection{Hashing}\label{classseq_1_1hold__any_autotoc_md2}
As said previously, \mbox{\hyperlink{classseq_1_1hold__any}{seq\+::hold\+\_\+any}} is hashable and a specialization for std\+::hash is provided. This feature is mandatory when inserting any objects into hash table based containers like std\+::unordered\+\_\+set. In order to store non hashable types (that do not specialize std\+::hash), you must specialize \mbox{\hyperlink{structseq_1_1is__hashable}{seq\+::is\+\_\+hashable}} type traits for this type to have seq\+::is\+\_\+hashable$<$\+Type$>$\+::value == false.

Another way to store non hashable types is to use \mbox{\hyperlink{namespaceseq_a2ec98840dd82cf3a3207402caa66c73f}{seq\+::nh\+\_\+any}} instead of \mbox{\hyperlink{namespaceseq_a24ef193cef19271b9b18ce6c74fc5b54}{seq\+::any}} which disable the hashing support. Attempting to call seq\+::hold\+\_\+any\+::hash() member on a non hashable type will throw a std\+::bad\+\_\+function\+\_\+call.

The specialization of std\+::hash for \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} is transparent to support heterogeneous lookup.

\hypertarget{classseq_1_1hold__any_autotoc_md3}{}\doxysubsubsection{Equality comparison}\label{classseq_1_1hold__any_autotoc_md3}
\mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} objects are equally comparable. This feature is mandatory when inserting any objects into hash table based containers like std\+::unordered\+\_\+set. Two \mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} are considered equal if\+:
\begin{DoxyItemize}
\item They are both empty.

\hypertarget{classseq_1_1hold__any_autotoc_md5}{}\doxysubsubsection{Small Buffer Optimization}\label{classseq_1_1hold__any_autotoc_md5}
\mbox{\hyperlink{classseq_1_1hold__any}{hold\+\_\+any}} provides a customizable Small Buffer Optimization (S\+BO) in order to avoid costly memory allocation for small types. The alignment of the small buffer can also be modified.
