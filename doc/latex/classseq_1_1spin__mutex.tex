\doxysubsection{Detailed Description}
\subsubsection*{template$<$unsigned spin\+\_\+count = 16U$>$\newline
class seq\+::spin\+\_\+mutex$<$ spin\+\_\+count $>$}

Mutex-\/like class combining a spinlock and std\+::mutex. 

\mbox{\hyperlink{classseq_1_1spin__mutex}{spin\+\_\+mutex}} is a kind of mutex that combines the speed of spin locking for uncontended scenarios and the C\+PU friendly wait of std\+::mutex.

On lock, \mbox{\hyperlink{classseq_1_1spin__mutex}{spin\+\_\+mutex}} first uses a spinning strategy for at most {\itshape spin\+\_\+count} cycles. If the spinlock fails, then the std\+::mutex is locked.

Note that the std\+::mutex is only allocated when needed, i.\+e. at the first failed attempt to lock the \mbox{\hyperlink{classseq_1_1spin__mutex}{spin\+\_\+mutex}} with spinning. Therefore, the size of \mbox{\hyperlink{classseq_1_1spin__mutex}{seq\+::spin\+\_\+mutex}} is only 16 bytes on 64 bits systems.

\mbox{\hyperlink{classseq_1_1spin__mutex}{spin\+\_\+mutex}} follows the Timed\+Mutex requirements.
