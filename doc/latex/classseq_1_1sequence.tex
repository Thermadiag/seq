\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T, class Allocator = std\+::allocator$<$\+T$>$, Layout\+Management layout = Optimize\+For\+Speed, bool Force\+Align64 = false$>$\newline
class seq\+::sequence$<$ T, Allocator, layout, Force\+Align64 $>$}

sequence is an ordered container supporting constant time insertion at both end and constant removal anywhere. 


\begin{DoxyTemplParams}{Template Parameters}
{\em T} & value type \\
\hline
{\em Allocator} & allocator type\\
\hline
\end{DoxyTemplParams}
The sequence container behaves like a hybrid version of std\+::deque and std\+::list. It provides\+:
\begin{DoxyItemize}
\item Constant time insertion at the back or the front using members \mbox{\hyperlink{classseq_1_1sequence_ab7a6dfc0af059f7d3c6dd0b255bea08f}{push\+\_\+back()}}, \mbox{\hyperlink{classseq_1_1sequence_ad849b79810bf1ad79912af6914834949}{emplace\+\_\+back()}}, \mbox{\hyperlink{classseq_1_1sequence_a1b1a3369b5343e9941d069ccc7cd4639}{push\+\_\+front()}} and \mbox{\hyperlink{classseq_1_1sequence_a70fffbaf997937a304178a81e9e92637}{emplace\+\_\+front()}}
\item Constant time removal of one or more elements with \mbox{\hyperlink{classseq_1_1sequence_a038a503f6e12ef433dd22463bf21cc0b}{erase()}}
\item Stability of references and iterators.
\end{DoxyItemize}

Unlike std\+::list, the sequence container does not provide insertion anywhere in the container. Instead, sequence provides unordered insertion through its member \mbox{\hyperlink{classseq_1_1sequence_a3a6276982c81ecc0cfc2b0f22a896a7a}{insert()}}, much like the \href{https://plflib.org/colony.htm}{\texttt{ plf\+::colony}} class. Unordered insertion is usually prefered to back or front insertion as it can reuse free slots created by \mbox{\hyperlink{classseq_1_1sequence_a038a503f6e12ef433dd22463bf21cc0b}{erase()}} and avoid potential allocations. In addition, its \mbox{\hyperlink{classseq_1_1sequence_a0c2005c28e8537de74b4c1e588180463}{sort()}}, \mbox{\hyperlink{classseq_1_1sequence_a5aaca3cd0a83c4637c28599ac55d70a0}{stable\+\_\+sort()}} and \mbox{\hyperlink{classseq_1_1sequence_a1bf494125859119c2748a7a5e59588fa}{shrink\+\_\+to\+\_\+fit()}} members do not preserve reference and iterator stability.

Its main advantages other std\+::list (and other containers) are\+:
\begin{DoxyItemize}
\item Insertion at both ends is faster than a std\+::deque or std\+::vector (without reserve)
\item Walking through the sequence with iterators is usually faster than walking through a std\+::deque
\item Sorting a sequence (with \mbox{\hyperlink{classseq_1_1sequence_a0c2005c28e8537de74b4c1e588180463}{sort()}} or \mbox{\hyperlink{classseq_1_1sequence_a5aaca3cd0a83c4637c28599ac55d70a0}{stable\+\_\+sort()}}) is usually as fast as sorting a std\+::deque. Note that \#sequence\+::sort() uses \href{https://github.com/orlp/pdqsort}{\texttt{ pdqsort}} from Orson Peters.
\item Its memory overhead is lower than a std\+::list\+: around 1 byte per element.
\end{DoxyItemize}

The sequence container is a perfect candidate for std\+::queue and std\+::stack. It is used by the seq library as the backend container for \mbox{\hyperlink{classseq_1_1ordered__set}{seq\+::ordered\+\_\+set}} and \mbox{\hyperlink{classseq_1_1ordered__map}{seq\+::ordered\+\_\+map}}.\hypertarget{classseq_1_1sequence_autotoc_md50}{}\doxysubsubsection{Technical description}\label{classseq_1_1sequence_autotoc_md50}
sequence container is implemented as a linked list of buckets. Each bucket holds (up to) 64 elements in a contiguous storage, and a 64 bits integer telling if a slot is empty or occupied.

In order to retrieve the index of the first (or last) used slot in a bucket, or to get the number of occupied slots, the sequence container uses OS intrinsics to scan the 64 bits integer. For instance, on Windows, {\itshape \+\_\+\+Bit\+Scan\+Forward64}, {\itshape \+\_\+\+Bit\+Scan\+Revers64} and {\itshape \+\_\+mm\+\_\+popcnt\+\_\+u64} are used. Removing an element from the sequence will set the corresponding bit to 0, inserting will set the bit to 1.

In addition, the sequence maintains another linked list of partially free buckets in order to perform fast unordered insertion using \mbox{\hyperlink{classseq_1_1sequence_a3a6276982c81ecc0cfc2b0f22a896a7a}{insert()}} member and therefore reuse slots previously deleted by \mbox{\hyperlink{classseq_1_1sequence_a038a503f6e12ef433dd22463bf21cc0b}{erase()}}.\hypertarget{classseq_1_1sequence_autotoc_md51}{}\doxysubsubsection{Iterators}\label{classseq_1_1sequence_autotoc_md51}
sequence iterator and const\+\_\+iterator are bidirectional iterators. However, they provide all the members to behave like random access iterator, except the std\+::random\+\_\+access\+\_\+iterator\+\_\+tag typedef.

Indeed, using {\itshape \mbox{\hyperlink{classseq_1_1detail_1_1sequence__iterator_a6eae5e249a991ab2b0ea792e6eb85b74}{iterator\+::operator+=}}} is, for instance, much faster than {\itshape std\+::advance} as it can skip whole buckets to reach the required location. Likewise subtracting 2 iterators is much faster than using {\itshape std\+::distance}.

sequence iterator also provide comparison operators $<$, $>$, $<$= and $>$=.\hypertarget{classseq_1_1sequence_autotoc_md52}{}\doxysubsubsection{Memory managment}\label{classseq_1_1sequence_autotoc_md52}
When using the {\itshape Optimize\+For\+Memory} flag, the sequence will allocate each bucket independently using the provided allocator.

When using the {\itshape Optimize\+For\+Speed} flag, the sequence will use a memory pool to allocate several buckets at once, still using the provided allocator. The memory pool uses a growing strategy to allocate more and more buckets based on a growth factor (S\+E\+Q\+\_\+\+G\+R\+O\+W\+\_\+\+F\+A\+C\+T\+OR define, defaulting to 1.\+6). The memory pool makes insertion tipically 50\% faster than the default allocation strategy, but will consume slightly more memory.

Whenever a bucket is empty (by calls to \mbox{\hyperlink{classseq_1_1sequence_a038a503f6e12ef433dd22463bf21cc0b}{erase()}}, \mbox{\hyperlink{classseq_1_1sequence_afa3e3d448af95c0bf3a7e897ae02f193}{clear()}}, \mbox{\hyperlink{classseq_1_1sequence_ac210c62cf9931e32cff1e0d0334e260a}{resize()}}, \mbox{\hyperlink{classseq_1_1sequence_a6e8496f2670bef2a4bd401f6671d8fc9}{resize\+\_\+front()}} or \mbox{\hyperlink{classseq_1_1sequence_a1bf494125859119c2748a7a5e59588fa}{shrink\+\_\+to\+\_\+fit()}}), the sequence releases the occupied memory.
