The memory module provides several classes to help for containers memory management. 

\hypertarget{group__memory_autotoc_md25}{}\doxysubsubsection{Allocators}\label{group__memory_autotoc_md25}
The memory module provides several types of S\+TL conforming allocators\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{classseq_1_1aligned__allocator}{seq\+::aligned\+\_\+allocator}}\+: allocator returning aligned memory
\item \mbox{\hyperlink{classseq_1_1external__allocator}{seq\+::external\+\_\+allocator}}\+: allocator using external allocation/deallocation functions
\item \mbox{\hyperlink{classseq_1_1object__allocator}{seq\+::object\+\_\+allocator}}\+: allocate memory using an object pool (see next section)
\end{DoxyItemize}

See class documentation for more details.\hypertarget{group__memory_autotoc_md26}{}\doxysubsubsection{Memory pools}\label{group__memory_autotoc_md26}

\begin{DoxyItemize}
\item \mbox{\hyperlink{classseq_1_1object__pool}{seq\+::object\+\_\+pool}}\+: standard memory pool for fast allocation of 1 or more objects of the same type, as well as std\+::unique\+\_\+ptr. Not thread safe.
\item \mbox{\hyperlink{classseq_1_1parallel__object__pool}{seq\+::parallel\+\_\+object\+\_\+pool}}\+: memory pool for fast allocation of 1 or more objects in a multi-\/threaded context. Uses an almost lock-\/free approach.
