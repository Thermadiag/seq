\doxysubsection{Detailed Description}
\subsubsection*{template$<$class T, class Allocator = std\+::allocator$<$\+T$>$, size\+\_\+t Align = 0, class object\+\_\+allocation = linear\+\_\+object\+\_\+allocation$<$1$>$, bool Generate\+Stats = false$>$\newline
class seq\+::parallel\+\_\+object\+\_\+pool$<$ T, Allocator, Align, object\+\_\+allocation, Generate\+Stats $>$}

Lock-\/free parallel object pool.

\mbox{\hyperlink{classseq_1_1parallel__object__pool}{seq\+::parallel\+\_\+object\+\_\+pool}} is very similar to \mbox{\hyperlink{classseq_1_1object__pool}{seq\+::object\+\_\+pool}}, but is a fully thread-\/safe alternative.

\mbox{\hyperlink{classseq_1_1parallel__object__pool}{seq\+::parallel\+\_\+object\+\_\+pool}} relies on independant thread-\/local memory blocks. Allocating is quite straightforward, and is very similar to using a thread\+\_\+local \mbox{\hyperlink{classseq_1_1object__pool}{object\+\_\+pool}}.

Deallocating is more complex, as a memory region can only be deallocated by the thread that created it (except if the thread was destroyed). Therefore, \mbox{\hyperlink{classseq_1_1parallel__object__pool}{parallel\+\_\+object\+\_\+pool}} uses the notion of deffered deletion\+:
\begin{DoxyItemize}
\item The deletion request is added to a list of requests
\item On further calls to parallel\+\_\+object\+\_\+pool\+::allocate() or \mbox{\hyperlink{classseq_1_1parallel__object__pool_a36e68df7db73bb812434ebb0c9988053}{parallel\+\_\+object\+\_\+pool\+::deallocate()}}, all pending deletion are processed for this thread.
\end{DoxyItemize}

When a thread is destroyed, several options are checked\+:
\begin{DoxyItemize}
\item All memory blocks belonging to this thread are gathered
\item Empty memory blocks are destroyed (\mbox{\hyperlink{classseq_1_1parallel__object__pool_a743de3a7f26697b9924b8283e9d133fa}{reclaim\+\_\+memory()}}==true) or added to a global list of free blocks (\mbox{\hyperlink{classseq_1_1parallel__object__pool_a743de3a7f26697b9924b8283e9d133fa}{reclaim\+\_\+memory()}}==false) that can be used later by any thread.
\item Non empty blocks are added to a list of blocks to \textquotesingle{}clean\textquotesingle{}. \mbox{\hyperlink{classseq_1_1parallel__object__pool}{parallel\+\_\+object\+\_\+pool}} will try to reuse blocks from the clean list whenever they become empty.
\end{DoxyItemize}

\mbox{\hyperlink{classseq_1_1parallel__object__pool}{parallel\+\_\+object\+\_\+pool}} is almost lock-\/free, and should be several times faster than the default malloc()/free() implementation in multi-\/threaded scenarios.

All members of \mbox{\hyperlink{classseq_1_1parallel__object__pool}{parallel\+\_\+object\+\_\+pool}} are thread safe.

For more details, see the documentation of \mbox{\hyperlink{classseq_1_1object__pool}{seq\+::object\+\_\+pool}}.

Set the reclaim\+\_\+memory flag. 

If false, the \mbox{\hyperlink{classseq_1_1object__pool}{object\+\_\+pool}} will not deallocate memory on calls to \mbox{\hyperlink{classseq_1_1parallel__object__pool_a36e68df7db73bb812434ebb0c9988053}{deallocate()}}. Instead, free memory blocks will be added to an internal list and reuse on calls to allocate(). This is the only way to move a memory block dedicated to an object count for another object count.

If true, calls to deallocate will deallocate any free block.
